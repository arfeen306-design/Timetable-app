# Pilot Readiness Checklist

Use this checklist to confirm the project is ready before pilot use in a real school.

## School setup
- [ ] School name and academic year set
- [ ] Days per week and periods per day configured
- [ ] Bell schedule reviewed (start time, period length)
- [ ] Zero period enabled if required
- [ ] Breaks configured (after which period, start/end times)
- [ ] Friday timing reviewed (if different from other days)

## Master data
- [ ] **Subjects** — All subjects added (or imported from Excel); names and codes correct
- [ ] **Classes** — All classes/sections added; grade, section, stream consistent; no duplicates
- [ ] **Teachers** — All teachers added; names and codes correct; max periods/day and max periods/week set
- [ ] **Rooms** — Classrooms/rooms added if room assignment is needed

## Lessons and constraints
- [ ] **Lessons** — All teacher–subject–class assignments entered; periods per week and duration set
- [ ] **Constraints** — Teacher/class/room availability reviewed (unavailable slots set where needed)

## Validation and generation
- [ ] **Validate Data** run on Generate page — no errors
- [ ] Warnings (if any) reviewed and accepted or fixed
- [ ] **Sample timetable generated successfully** (click Generate and wait for completion)
- [ ] **Conflict report** — After generation, check output: no conflicts (or conflicts explained)
- [ ] **Unscheduled lessons** — Check output: all lessons scheduled or shortfall understood
- [ ] **Teacher workload** — Review teacher timetables; total weekly periods match expectations

## Exports and print
- [ ] **Class timetable** — Export to Excel/PDF; times and breaks correct; readable
- [ ] **Teacher timetable** — Export to Excel/PDF; workload total correct; Friday timing correct if different
- [ ] **Master timetable** — Export reviewed; classes in sensible order; print-friendly if needed

## Before going live
- [ ] One real-school test dataset imported and validated
- [ ] Generation run with real data; results and exports checked
- [ ] Known limitations (see KNOWN_LIMITATIONS.md) understood and accepted

---

**How to use:** Work through each section in order. If any item is not checked, fix or confirm before pilot use. Keep this checklist for each pilot school or academic term.
